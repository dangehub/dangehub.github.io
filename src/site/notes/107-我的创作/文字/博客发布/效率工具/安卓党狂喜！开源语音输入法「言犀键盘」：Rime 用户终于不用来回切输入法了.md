作为安卓 + Rime 输入法深度用户，我之前一直被语音输入搞得很崩溃 —— 想用语音就得切到讯飞，输完再切回 Rime，来回切换又麻烦又打断思路。直到发现 Linux Do 论坛用户开发的「言犀键盘」，不仅解决了我的痛点，体验甚至追上了 iOS 上的标杆软件，今天必须认真安利给大家！

先跟用过 iOS 的朋友提一嘴，好的语音输入工具该有的样子，比如 Spokenly 那种：能选本地 / 在线模型、支持 LLM 后处理文本（去语气词、口语转书面）。但 iOS 第三方键盘权限太死，用的时候得跳应用、还会占语音通道，听音乐时语音输入甚至会降音质，而且永远没法和自己常用的键盘共存。

而安卓的「言犀键盘」，直接把这些问题全解决了，还加了不少惊喜功能：

![assets/7ac7117e7bc312ca7762cd8344531bdef02bd790.png](/img/user/107-%E6%88%91%E7%9A%84%E5%88%9B%E4%BD%9C/%E6%96%87%E5%AD%97/%E5%8D%9A%E5%AE%A2%E5%8F%91%E5%B8%83/%E6%95%88%E7%8E%87%E5%B7%A5%E5%85%B7/assets/7ac7117e7bc312ca7762cd8344531bdef02bd790.png)

1. **悬浮球 + 任意键盘共存，Rime 党狂喜**  不用切换输入法！悬浮球只在有输入框时出现，点一下就能语音输入，输完直接用自己熟悉的键盘（比如 Rime）微调。我现在写文档都是先语音输大段内容，再用 Rime 改错别字，效率直接翻倍。当然言犀键盘也支持键盘模式。

2. **权限友好，用着不折腾**  不用跳转应用加载模型，也不占用语音通道。之前用 iOS 软件时，听着歌语音输入就会降音质，用言犀键盘完全没这问题，后台放歌、前台语音输入两不误。
3. **LLM 后处理 + 高自定义， accuracy 拉满**  我目前用的是豆包语音模型，准确率很高，配合 LLM 后处理，基本不用手动改。而且悬浮球上能直接开关 LLM 优化、切换提示词或 API 供应商，想自定义功能也不用翻复杂设置。
4. **集成开源的剪贴板同步项目** 甚至还能把手机作为全终端的语音输入工具
5. **开源免费，安全感拉满**  毕竟是开源项目，代码都在 GitHub 上（后附链接），不用担心隐私问题，有能力的朋友还能自己改功能，这点比很多闭源软件靠谱多了。
    

最后再总结下，如果你是安卓用户，尤其是习惯用 Rime 等小众键盘、需要高频语音输入的人，「言犀键盘」绝对值得试！直接去 GitHub（[BryceWG/LexiSharp-Keyboard](https://github.com/BryceWG/LexiSharp-Keyboard)）就能下载，亲测用下来，比我之前试过的所有安卓语音输入法都顺手～
